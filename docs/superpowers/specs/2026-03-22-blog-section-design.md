# Blog Section — Design Spec

## Overview

Add a blog to the personal portfolio site focused on exploring AI implications, technologies, industry developments, and research. Mix of short takes (300-500 words) and medium essays (800-1200 words), published every 2-3 days. Includes a cowork prompt for automated draft generation.

## Content Architecture

### MDX Files

Blog posts stored as `.mdx` files in `src/content/blog/`. Each file is a self-contained post with frontmatter metadata.

```
src/content/blog/
├── 2026-03-22-agents-are-eating-software.mdx
├── 2026-03-24-claude-4-what-changed.mdx
└── ...
```

### Frontmatter Schema

```yaml
---
title: "Agents Are Eating Software"
date: "2026-03-22"
description: "Why agentic systems are replacing traditional SaaS workflows faster than anyone expected."
tags: ["agents", "industry"]
readTime: "4 min read"
---
```

### Draft Files

Drafts generated by cowork go to `src/content/blog/drafts/` and are NOT rendered on the site. They only become posts when moved to `src/content/blog/` via a merged PR.

## Routing

### Build Configuration

Remove `output: "export"` from `next.config.ts`. Replace with `generateStaticParams` in blog routes so all pages are still statically generated at build time. GitHub Pages deployment workflow remains unchanged — `npm run build` still produces static output to `out/`.

**Update:** Keep static output by using `output: "export"` with `generateStaticParams` — Next.js static export supports dynamic routes when `generateStaticParams` is provided. No config change needed.

### Routes

- **`/blog`** — blog index page listing all posts
- **`/blog/[slug]`** — individual blog post page
- **Slug derivation:** filename without date prefix and `.mdx` extension (e.g., `2026-03-22-agents-are-eating-software.mdx` → `agents-are-eating-software`)

## Blog UI

### Blog Index (`/blog`)

- Same warm + Playfair/Inter style as rest of site
- Nav bar visible (same floating pill nav, add "Blog" link)
- Page title: "Blog" in Playfair serif
- Subtitle: "Exploring AI — new developments, research, and where things are heading."
- Posts listed newest first, vertically stacked
- Each post entry:
  - Title (Playfair serif, clickable link to post)
  - Date + read time in muted text
  - Description (1-2 sentences)
  - Tags as small pills (same style as project tech tags)
- No pagination initially — list all posts (can add later if >20 posts)

### Blog Post (`/blog/[slug]`)

- Back link: "← Back to blog" at top
- Post header:
  - Title in Playfair serif (32-36px)
  - Date + read time in muted text
  - Tags as pills
- Post body:
  - MDX rendered with Tailwind Typography (`@tailwindcss/typography`) for prose styling
  - Max-width ~680px (`max-w-prose`) for comfortable reading
  - Headings, paragraphs, code blocks, links, lists, blockquotes all styled via typography plugin
- Footer: "← Back to blog" link at bottom

## Content Utilities

### Blog data helpers (`src/lib/blog.ts`)

Utility functions for reading and parsing MDX files:
- `getAllPosts()` — reads all `.mdx` files from `src/content/blog/` (not drafts), parses frontmatter, returns sorted by date descending
- `getPostBySlug(slug)` — reads a single post by slug, returns frontmatter + compiled MDX content
- `getLatestPosts(n)` — returns the n most recent posts

Uses `gray-matter` for frontmatter parsing and `next-mdx-remote` for MDX compilation.

## Tech Additions

- **`@tailwindcss/typography`** — prose styling for blog posts
- **`gray-matter`** — frontmatter parsing
- **`next-mdx-remote`** — MDX compilation for static rendering

## Cowork Prompt

Provide a ready-to-paste prompt for Claude cowork scheduled tasks that:
- Picks a recent/trending AI topic (model releases, papers, industry news, research)
- Writes a draft as an MDX file with proper frontmatter
- Commits to a new branch `blog/draft-YYYY-MM-DD-slug`
- Opens a PR with a summary of the draft
- Targets the `aineshm/aineshm.github.io` repo

The prompt will be saved to `docs/cowork-blog-prompt.md` for reference.

## Accessibility

- Blog post pages maintain all existing accessibility standards
- Prose content has proper heading hierarchy (h2, h3 — no h1 in body, title is h1)
- Links in prose are visually distinct (underlined, cardinal red on hover)
- Code blocks have sufficient contrast

## Responsive

- Blog index and post pages follow same breakpoints as main site
- Post body max-width `max-w-prose` (~680px) centered on all screen sizes
- On mobile, full-width with horizontal padding
